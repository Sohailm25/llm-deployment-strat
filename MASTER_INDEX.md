## Document Standards

### Structure
Each document follows this standard structure:
1. **Title Page**: Document ID, title, version, date, author
2. **Executive Summary**: 1-page overview for executives
3. **Prerequisites**: Required knowledge, tools, access
4. **Main Content**: Detailed guidance with sections
5. **Implementation Guide**: Step-by-step procedures
6. **Code Examples**: Working code samples
7. **Configuration Templates**: Ready-to-use configs
8. **Troubleshooting**: Common issues and solutions
9. **Appendices**: Reference materials, checklists
10. **Glossary**: Key terms defined
11. **References**: External resources, further reading

### Quality Criteria
- Self-contained: No assumed prior knowledge beyond prerequisites
- Actionable: Clear steps to implement
- Complete: Covers the full topic scope
- Current: December 2025 best practices
- Tested: Code examples verified to work

### Cross-References
Documents reference related documents using format: `[Doc X.Y: Title]`
